## Change Tested

This run tests the roll-forward repair: when the common stress trigger fires on
a live 7DTE SPXW iron condor, the strategy closes the current condor and re-enters
a replacement condor at a fresh valid baseline-style expiry from the trigger date.

## Method

Ran `qc/spxw_7dte_rollout` in QuantConnect Cloud using the committed code version
`ba59b758aadb9e38b67879e0f9faab59bbca8fa1`. Metrics in this record are taken from
the saved results payload at
`research_log/results/e015_h015_spxw_7dte_rollout.json`.

## Interpretation

The rollout result is the most balanced repair outcome so far. Compared with
hard exit, it gave back some additional drawdown protection, but it improved
trade-level economics materially: positive expectancy, a 1.21 profit-loss ratio,
and a 50% win rate. Compared with same-expiry recenter, rollout was clearly
better on net loss, drawdown, Sharpe, Sortino, and expectancy. This supports
the core `h015` idea that adding time after trigger helps more than simply
resetting strikes on the same expiry.

## Artifacts And Caveats

- Suspicious artifacts: None reported in the saved QuantConnect payload.
- Data issues: None reported in the saved QuantConnect payload.
- Execution anomalies: Order count remained very high at `11,320`, though fees (`$1,056.00`) were lower than the recenter variant.
