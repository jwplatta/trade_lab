## Change Tested

This run tests the same-expiry recenter repair: when the common stress trigger
fires on a live 7DTE SPXW iron condor, the strategy closes the current condor
and re-enters a new symmetric condor on the same expiry.

## Method

Ran `qc/spxw_7dte_recenter` in QuantConnect Cloud using the committed code
version `7870c49669dd5203df24e6d89ad1b700f53899e4`. Metrics in this record are
taken from the saved results payload at
`research_log/results/e014_h014_spxw_7dte_recenter.json`.

## Interpretation

The recenter rule helped relative to the unadjusted baseline, confirming that
removing the original tested strikes has some value. But it did not outperform
hard exit, which kept both drawdown and net loss materially lower. That result
supports the concern in `h014`: resetting the condor on the same expiry appears
to reintroduce risk too quickly, leaving the strategy exposed to continued
adverse path behavior while also increasing execution load.

## Artifacts And Caveats

- Suspicious artifacts: None reported in the saved QuantConnect payload.
- Data issues: None reported in the saved QuantConnect payload.
- Execution anomalies: Order count rose to `11,472` and fees to `$1,448.00`, making recenter the most execution-heavy variant tested so far.

Run `qc/spxw_7dte_rollout` as `e015` to test whether adding time after trigger
retains more of the hard-exit drawdown benefit than same-expiry recentering.
